为什么要有进程这个概念

需求1：单核处理器上在同一时间运行不同的程序，（例如边听歌边写代码）。

  • 时分复用：只要速度够快，就能实现影分身
grep 'CONFIG_HZ=' /boot/config-$(uname -r)

需求2: 安全问题

  • 随意更改内存中的其他程序的代码和数据
  • 随意读写文件系统中的敏感数据(密码)
  • 写程序位置偏移

计算机科学领域的任何问题都可以通过增加一个间接的中间层来解决

进程 API

  • 进程的创建
    • fork/execve (*NIX)
  • 进程的销毁
    • exit (*NIX)
  • 与 OS 交互
    • syscall
    • ausyscall --dump | less
  • 与其他进程交互
    • wait, kill(信号) .etc

信号

事件处理程序 <==> 用户态中断处理程序

  • 屏蔽: sigpromask (via set blocked flags)
  • 解除: sigpromask (via set unblocked flags)
  • 设置处理程序: sigaction (ucontext)
  • Supporting functions
    • sigemptyset – Create empty set
    • sigfillset – Add every signal number to set
    • sigaddset – Add signal number to set
    • sigdelset – Delete signal number from set
  • AA类型死锁
    • deadlock.c: printf -> vfprintf -> _IO_flockfile(s)
    • 解决方法: (1) 开关中断 (2)不用全局变量(没有锁)
